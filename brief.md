# OS Playbook / project brief (regeneration / alignment)

Use this file when asking an AI or stakeholder to **extend** the repo without re-explaining context.

## Purpose

Internal resource for Jobber Onboarding Specialists: **routing**, **module topic cards** (Help Center–anchored), and a **per-SP journey record**; not a customer LMS, not step-by-step duplication of Help Center, not wiki sprawl.

## Success (SP)

Value, confidence, calm. Baseline: **workflow fit** (their variant of request → quote → job → invoice), **team on jobs** when relevant, **active client communications**. Engagement: **real data**, **payments** (including non–Jobber Payments), supporting signals (products, website, GBP) as available.

## Constraints

- English v1. One OS per SP (no handoff workflow). ~30 min sessions, ~15 min admin. ~75–90 day window.
- **Help Center = how; playbook = when, why, enough.**

## Architecture

1. **Playbook:** `playbook/`, `modules/`, routing, M01–M12 cards.  
2. **Journey record:** `journey-record/TEMPLATE.md` per engagement.

## Modules

M01 Discovery → M02 Setup → M03 Payments → M04 Phone (optional) → M05 Workflow → M06–M12 expansion. Each card: outcome, focus areas, HC bundle (TODO links), failure modes, skip/compress/defer, observables, next steps.

## Maintenance

Replace all `TODO` Help Center URLs with current articles monthly or when product changes.
