# M12: Advanced topics and integrations

**Module ID:** `M12`  
**Typical phase:** Optimization  
**Default prerequisites:** Core operations stable; **Eric / Paula** or explicit integration need

### Learning outcome

**Integrations and advanced workflows** are chosen **deliberately** (accounting sync, third-party tools, API/Zapier-style flows) without derailing day-to-day Jobber use.

### Focus areas (30 minutes)

- **Discover need:** QuickBooks / Xero / other, **one** integration thread per session.  
- **Data direction and reconciliation** expectations (high level).  
- **Zapier / automation** only if SP has **owner or technical** bandwidth.  
- **Escalate** to Support or specialist resources when appropriate; don’t improvise API security.

### Common failure modes

- **Integration before** invoice and job discipline.  
- **Broken sync** erodes trust, set **success criteria** and **rollback** expectations.

### Skip if …

- **No integration need** in the next 90 days, note “not applicable.”

### Compress if …

- **Accounting only**: single setup checklist from HC.

### Defer if …

- **Foundation shaky**: defer until evidence improves (see [`playbook/how-to-use.md`](../playbook/how-to-use.md) evidence hierarchy).

### Done looks like (observable)

- **One integration** **live** with **owner knowing** who to call when it breaks OR explicit **phase-2** plan with date.

### Natural next modules

- **Graduation:** move to **success / check-in** rhythm per your org, not new modules.
