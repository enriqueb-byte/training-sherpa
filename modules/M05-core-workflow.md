# M05: Core workflow

**Module ID:** `M05`  
**Typical phase:** Foundation → Workflow  
**Default prerequisites:** M02; M03 started or explicitly deferred

### Learning outcome

The SP can move **real work** through Jobber using **their** lifecycle: typically request → quote → job → invoice → paid, **including valid shortcuts** (straight to job, invoicing elsewhere).

### Focus areas (30 minutes)

- Draw **their** chain on a napkin, match Jobber vocabulary to **their** business.  
- **Requests / leads:** how work enters (phone, form, repeat client).  
- **Quotes / estimates:** when they quote vs. book.  
- **Jobs:** schedule, assign, complete, **team assign** when relevant.  
- **Invoices + payment:** tie to **M03**; if invoicing is **external**, clarify what **must** live in Jobber for visibility.  
- **Client Hub** entry point at high level, deeper in **M09**.

### Common failure modes

- **Quotes that never become jobs**: stuck pipeline.  
- **Invoices disconnected** from completed work.  
- **Owner never repeats** the loop, only staff uses Jobber.

### Skip if …

- **Heavy invoice history already in Jobber**, **compress** to pipeline + job assignment + one gap.

### Compress if …

- They are **doing** the workflow but messy, fix **one** broken transition.

### Defer if …

- **Risk flags** red (imports chaos, owner non-use), stabilize inputs and login first.

### Done looks like (observable)

- At least **one full loop** completed or credibly scheduled in their real account.  
- **Team on jobs** if they have field staff.  
- You can name **their** variant in one sentence on the journey record.
