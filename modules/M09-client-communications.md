# M09: Client communications

**Module ID:** `M09`  
**Typical phase:** Expansion  
**Default prerequisites:** Core workflow recurring enough that messaging matters

### Learning outcome

**Outbound and automated** client touchpoints are **intentional** (quotes, booking reminders, job follow-ups) without overwhelming the SP.

### Focus areas (30 minutes)

- **Client communications** area: what Jobber sends by default vs. customizable.  
- **Templates**: tone and brand; edit **1–2** high-volume emails/SMS in session.  
- **Automations** at appropriate depth for persona (Sam: fewer; Paula: more optimization).  
- Tie to **Client Hub** experience, consistency with **M05**.

### Common failure modes

- **Template theater**: beautiful emails, no jobs.  
- **Too many automations** before team follows through.

### Skip if …

- **Heavy customization already live**: review gaps only.

### Compress if …

- Fix **one** broken touchpoint (e.g., quote follow-up only).

### Defer if …

- **Workflow still shaky**: automating bad process.

### Done looks like (observable)

- **Observable** client messages going out; responses or confirmations happening; SP knows where to **edit** next time.
